# OpenAI Hackathon Submission Checklist

## Required before submission

- [ ] Demo video is under three minutes, public on YouTube, and its URL is correct in Devpost.
- [ ] Voiceover explains the product, how Codex was used, and how GPT-5.6 was used.
- [ ] The `/feedback` Codex session ID is retrieved and entered in the form.
- [ ] The private code repository is shared with Devpost and OpenAI when required.
- [ ] The README includes setup instructions and an accurate explanation of Codex and GPT-5.6 usage.
- [ ] Plugin or developer-tool submissions include installation and judge testing instructions.
- [ ] Every team member has accepted their invitation.
- [ ] A category is selected.
- [ ] The submission is published, not saved as a draft.

## OpsControl judge access

- Live app: https://opscontrol.streamlit.app/
- Credentials: none required for the deterministic demo.
- Fast test: reset the desk, replay the Savannah storm, inspect `OPS-40045-A`, review a low-confidence item, then replay duplicates.

## Video finishing notes

- Cut loading, typing, silence, and restarts before recording the final edit.
- Rehearse the narration once; direct language reads better than filler.
- Mild playback speed increases can help if the explanation remains clear.
- Use AI to assist narration or structure, but edit the final description so it accurately reflects the product.
- Keep the product name intentional and consistent across the video, Devpost, README, and repository.
